# UMA Specification v1.1
Unified Meta‑Architecture — Conceptual Specification

## 1. Purpose
UMA defines a clear, multi‑plane way to describe complex systems without collapsing structure, governance, ethics, or documentation into each other.  
It is a conceptual mapping tool, not a governance system or operational framework.

## 2. Core Principles
- **Plane Separation:** Each plane (structural, governance, ethical, documentation) remains distinct.  
- **Non‑Reification:** UMA is a model, not a truth.  
- **Ethical Primacy:** All interpretation is grounded in the Sādhana stance.  
- **Clarity Over Complexity:** No unnecessary abstraction or mystification.

## 3. The Four Planes

### 3.1 Structural Plane
Describes the “shape” of a system:
- components  
- relationships  
- flows  
- boundaries  
- maps and diagrams  

This plane answers: **“What is here?”**

### 3.2 Governance Plane
Describes constraints and permissions:
- rules  
- invariants  
- boundaries  
- allowed / disallowed transitions  

This plane answers: **“What is allowed?”**

### 3.3 Ethical Plane
Describes the stance from which the system is interpreted:
- humility  
- non‑authority  
- non‑coercion  
- clarity  
- non‑reification  

This plane answers: **“How should we relate to this?”**

### 3.4 Documentation Plane
Describes how components are recorded:
- MPAM attachments  
- multi‑plane notes  
- examples  
- clarifications  

This plane answers: **“How do we describe this without mixing layers?”**

## 4. Allowed Operations

### 4.1 Mapping
Creating structural diagrams or conceptual layouts.

### 4.2 Annotation
Adding MPAM attachments to clarify components.

### 4.3 Boundary Definition
Specifying governance constraints in conceptual form.

### 4.4 Ethical Framing
Applying Sādhana to maintain humility and non‑authority.

## 5. Prohibited Uses
UMA must not be used for:
- real‑world governance  
- psychological authority  
- institutional control  
- metaphysical claims  
- coercive or manipulative systems  

UMA is conceptual only.

## 6. Versioning
Version numbers indicate conceptual refinement, not software releases.  
v1.1 clarifies plane boundaries and ethical commitments.

## 7. One‑Sentence Summary
UMA is a clean, multi‑plane way to describe systems without collapsing structure, governance, ethics, or documentation.
