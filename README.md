# Unified Meta-Architecture (UMA), Sādhana, and MPAM

This repository contains three tightly related pieces:

- **UMA (Unified Meta-Architecture):**  
  A conceptual, multi-plane meta-architecture for thinking about systems, structure, and governance.

- **Sādhana:**  
  An ethical, non-dual stance for how we relate to systems, power, and meaning.

- **MPAM (Multi-Plane Attachment Method):**  
  A documentation method for architectures that span multiple planes (structural, governance, developmental, dynamic, integrative).

This is **conceptual and exploratory work**.  
It is meant for thinking, modeling, and learning—not for running real-world institutions or exercising authority.

## Ethical Core

This repository is grounded in three simple vows:

- **No architecture without ethics.**  
- **No governance without humility.**  
- **No map mistaken for the territory.**

The intent of this work is to explore safer, clearer ways of thinking about complex systems, not to create a new authority or doctrine.

## Ethical Use & Limits

This work is intended for:
- conceptual exploration
- research and learning
- documentation design
- thinking about systems and governance

This work is **not** intended for:
- real-world governance or authority
- legal, medical, or psychological decision-making
- coercive, manipulative, or deceptive systems

High-stakes use requires qualified human oversight.

## Risks & Misinterpretations

Because this work deals with systems, governance, and meaning, it can be misunderstood in several ways. These risks are named explicitly to prevent misuse:

- **Reification:**  
  Treating UMA, MPAM, or Sādhana as absolute truth rather than conceptual tools.

- **Authority Inflation:**  
  Using the language of governance or ethics to claim power, status, or control.

- **Overreach:**  
  Applying this work to real-world institutions, legal systems, or vulnerable populations.

- **Cosmologizing:**  
  Treating Sādhana as a belief system, metaphysics, or spiritual doctrine.

This repository is conceptual and exploratory.  
It is not a governance system, not a court, and not a source of authority.

## How to Read This Repository

This repository is organized across multiple conceptual planes. Each plane has its own purpose and should not be collapsed into the others.

- **Structural Plane (Atlas / docs/):**  
  The maps, grids, and positional structures. This is the “layout” of the architecture.

- **Governance Plane (UMA Spec / uma-spec/):**  
  The rules, boundaries, and permissions. This is the “constitution” of the architecture.

- **Ethical Plane (Sādhana):**  
  A stance, not a module. It guides how the work should be approached, not how systems should behave.

- **Documentation Plane (MPAM / attachments/):**  
  Multi-plane attachments that describe specific components without mixing layers.

- **Examples Plane (examples/):**  
  Low-stakes demonstrations of how UMA + MPAM can be used conceptually.

When navigating this repository, treat each plane as distinct.  
Do not interpret any single file as “the whole system.”

## How to Engage Critically

This work is meant to be examined, questioned, and challenged.  
Critical engagement strengthens clarity and prevents misuse.

When reading or using this repository:

- **Assume fallibility:**  
  Treat every model, map, or specification as provisional.

- **Interrogate boundaries:**  
  Ask whether a concept belongs to the structural, governance, ethical, or documentation plane.

- **Challenge assumptions:**  
  If something appears universal, necessary, or absolute, examine it more closely.

- **Avoid deference:**  
  UMA, MPAM, and Sādhana are tools for thinking, not authorities to obey.

- **Offer alternatives:**  
  If you disagree with a mapping or structure, propose a different one.  
  Disagreement is part of the intended use.

This repository grows through critique, not consensus.

## Contributing (Ethical Guidelines)

Contributions are welcome, but they must align with the ethical stance and boundaries of this project.

Before contributing, please ensure:

- **Conceptual, not operational:**  
  Contributions should expand clarity, structure, or understanding — not create real-world governance systems.

- **No authority inflation:**

## Glossary (Plain Definitions)

**UMA (Unified Meta‑Architecture)**  
A conceptual way of mapping how different parts of a system relate across multiple planes.

**Sādhana**  
An ethical stance of humility, non‑duality, and non‑authority.  
It guides how we approach systems, not how systems should behave.

**MPAM (Multi‑Plane Attachment Method)**  
A documentation method that keeps different layers of a system separate so they don’t collapse into each other.

**Plane**  
A distinct layer of meaning or structure (e.g., structural, governance, ethical, documentation).

**Structural Plane**  
The “map” — shapes, layouts, relationships, and positions.

**Governance Plane**  
The “rules” — boundaries, permissions, constraints, and safeguards.

**Ethical Plane**  
The stance — humility, non‑authority, and clarity.

**Documentation Plane**  
The attachments — how components are described without mixing planes.

**Reification**  
Treating a model as if it were reality.

**Authority Inflation**  
When a conceptual tool is mistaken for a source of power or control.

**Overreach**  
Using a conceptual tool in a real‑world, high‑stakes context where it doesn’t belong.
